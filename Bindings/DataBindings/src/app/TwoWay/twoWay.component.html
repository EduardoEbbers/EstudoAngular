Interpretação da Estrutura Básica

One-way data binding são recursos usados em templates HTML do Angular, para gerar conteúdos ou modificar comportamentos através da interação com o usuário.

Possui o fluxo de dados em uma direção (Ver Figura 1):

  • Binding de Dados (Data Bindings)

    Os dados fluem do componente para o template.

  • Binding de Eventos (Event Bindings)

    Os dados fluem do template para o componente.

A estrutura básica, à esquerda da Figura 1, é dividida em quatro partes:

  • Elemento hospedeiro (Host Element)

    Elemento HTML ou tag HTML do qual vai mudar a aparência, conteúdo ou comportamento.

    Obs.: Data Bindings pode ser aplicado a qualquer elemento host e um elemento host pode ter múltiplos data bindings.

  • Colchetes (Brackets)

    Colchetes (ou Parênteses) envolvem o alvo denotando avaliar uma expressão que passa o resultado para o alvo, do qual faz mudança no elemento host.

    Colchetes indica ser um data binding.

    Parênteses indica ser um event binding.

    Obs.: Pode-se também usar dupla chaves, visto em string interpolation binding, e uma combinação de colchetes envolvendo parênteses chamado banana-in-box, visto em two-way bindings .

    Obs.: Omitindo os colchetes ou parênteses a expressão não será avaliada, passando a ser considerada um valor literal .

  • Alvo (Target)

    É aquele que vai especificar o que o binding vai fazer, sendo:

    • Bindings de Diretiva (Directive Bindings)

      Podem ser fornecidos pelo Angular (built-in directives) prefixados com ng ou personalizados.

      São divididos em:

      • Diretiva NgModel
  • Expressão (Expression)

    Fragmento de código analisado através de propriedades e métodos do componente (ou escrito inline no template).

    O retorno do componente para a expressão no template modifica a propriedade, dando funcionalidade ao binding que modifica o elemento host.

    Obs.: Na expressão pode-se misturar métodos e propriedades do componente com valor literal .

    Obs.: Dependendo da complexidade da lógica não é aconselhável escrever inline no template.

    Obs.: Quando se tem mais de um valor para se apresentar na expressão, deve-se separar por espaços.
