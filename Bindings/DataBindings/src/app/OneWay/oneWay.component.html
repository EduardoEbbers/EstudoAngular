Estrutura Básica

- interpretacao: se a quantidade de produtos for igual a 5 (true) vai ser a expressao sera bg-success, se nao (false) vai ser bg-warning - qdo se tem mais de um valor para se apresentar na expressao, deve-se separar por espacos

A estrutura básica, acima, é dividida em 4 partes:

  • Elemento hospedeiro (host element)

    Elemento HTML ou tag HTML do qual vai mudar a aparência, conteúdo ou comportamento.

    Data Bindings pode ser aplicado a qualquer elemento host e um elemento host pode ter múltiplos data bindings.

  • Colchetes ou Parênteses

    Colchetes ou Parênteses indicam que o data bindings possui uma expressão para ser avaliada.

    Colchetes indicam que os dados fluem do componente para o template (data bindings).

    Parênteses indicam que os dados fluem do template para o componente (event bindings).

    • Obs 1.: pode-se usar string interpolation bindings visto futuramente

    • Obs 2.: Omitindo o colchetes ou parênteses a expressão não será avaliada passando a ser considerada um valor literal.

    • Obs 3.: pode-se usar "banana-in-box" visto futuramente

  • Alvo (target)

    É aquele que vai ser atingido pelo binding, podendo ser de 2 tipos:

    • Bindings de Propriedade (Property Bindings)

      São divididos em:

      • Binding de Propriedade Padrão (Standard Property Binding)
      • Binding de Interpolação de String (String Interpolation Binding)
      • Binding de Atributo (Attribute Binding)
      • Binding de Class (Class Binding)
      • Binding de Estilo (Style Binding)
      • Binding de Evento (Event Binding)
    • Bindings de Diretiva (Directive Bindings)

      Podem ser fornecidos pelo Angular (built-in directives), prefixados com ng, ou personalizados

      São divididos em:

      • Diretiva ngClass
      • Diretiva ngStyle
      • Diretiva ngIf
      • Diretiva ngFor
      • Diretiva ngSwitch/ngSwitchCase/ngSwitchDefault
      • Diretiva ngTemplateOutlet

  • Expressão

    Fragmento de código JavaScript analisado pelo componente para ser um valor do alvo, tendo acesso às propriedades e métodos definidos no componente.

    Pode-se misturar metodos e propriedades com um valor string literal na expressão
