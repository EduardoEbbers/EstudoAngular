Interpretação da Estrutura Básica

À esquerda da Figura 1, temos o template (ou documento HTML) contendo a representação do elemento host (ou tag div) e o data binding representado por colchetes (ou parênteses) que envolve o alvo (diretiva ngClass). A diretiva ngClass representa a propriedade class que, dependendo do retorno da expressão (método getClasses() do componente), faz modificação no elemento host div.

À direita da Figura 1, temos o componente do template contendo o método getClasses(), do qual terá comunicação com o template através do retorno deste método.

Dependendo do fluxo de dados temos data binding ou event binding caracterizado pelo colchetes (ou parênteses).

Assim na Figura 1 temos um data binding (representado por colchetes) em que primeiro o Angular avalia a expressão da diretiva ngClass que é o método getClasses() do componente.

O método getClasses() do componente através do operador ternário analisa se a quantidade de total de produtos é igual a cinco, retornando um string "bg-success" ou "bg-warning".

Fluxograma do Método getClasses()

Como mostra o fluxograma da Figura 2 acima, caso true (ou seja, a quantidade de produtos é igual a 5) o retorno será "bg-success" que colore o plano de fundo do elemento host div de verde; caso false (ou seja, a quantidade de produtos não é igual a 5) o retorno será bg-warning que colore o plano de fundo do elemento host div de amarelo.

Após o retorno do método do componente, o Angular seta a expressão do template que modifica a propriedade class (através da diretiva ngClass) colorindo o plano de fundo (bg de brackground-color) do elemento host div de verde (success) ou amarelo (warning).

One-way data binding são recursos usados em templates HTML do Angular,
para gerar conteúdos ou modificar comportamentos através da interação com o usuário.

Possui o fluxo de dados em uma direção (Ver Figura 1):

  • Binding de Dados (Data Bindings)

    Os dados fluem do componente para o template.

  • Binding de Eventos (Event Bindings)

    Os dados fluem do template para o componente.

A estrutura básica, à esquerda da Figura 1, é dividida em quatro partes:

  • Elemento hospedeiro (Host Element)

    Elemento HTML ou tag HTML do qual vai mudar a aparência, conteúdo ou comportamento.

    Obs.: Data Bindings pode ser aplicado a qualquer elemento host e um elemento host pode ter múltiplos data bindings.

  • Colchetes (Brackets)

    Colchetes (ou Parênteses) envolvem o alvo denotando avaliar uma expressão que passa o resultado para o alvo, do qual faz mudança no elemento host.

    Colchetes indica ser um data binding.

    Parênteses indica ser um event binding.

    Obs.: Pode-se também usar dupla chaves, visto em string interpolation binding, e uma combinação de colchetes envolvendo parênteses chamado banana-in-box, visto em two-way bindings .

    Obs.: Omitindo os colchetes ou parênteses a expressão não será avaliada, passando a ser considerada um valor literal .

  • Alvo (Target)

    É aquele que vai especificar o que o binding vai fazer, podendo ser de 2 tipos:

    • Bindings de Propriedade (Property Bindings)

      São divididos em:

      • Binding de Propriedade Padrão (Standard Property Binding)
      • Binding de Interpolação de String (String Interpolation Binding)
      • Binding de Atributo (Attribute Binding)
      • Binding de Classe (Class Binding)
      • Binding de Estilo (Style Binding)
      • Binding de Evento (Event Binding)
    • Bindings de Diretiva (Directive Bindings)

      Podem ser fornecidos pelo Angular (built-in directives) prefixados com ng ou personalizados.

      São divididos em:

      • Diretiva NgClass
      • Diretiva NgStyle
      • Diretiva NgIf
      • Diretiva NgFor
      • Diretiva NgSwitch/NgSwitchCase/NgSwitchDefault
      • Diretiva NgTemplateOutlet
  • Expressão (Expression)

    Fragmento de código analisado através de propriedades e métodos do componente (ou escrito inline no template).

    O retorno do componente para a expressão no template modifica a propriedade, dando funcionalidade ao binding que modifica o elemento host.

    Obs.: Na expressão pode-se misturar métodos e propriedades do componente com valor literal .

    Obs.: Dependendo da complexidade da lógica não é aconselhável escrever inline no template.

    Obs.: Quando se tem mais de um valor para se apresentar na expressão, deve-se separar por espaços.
