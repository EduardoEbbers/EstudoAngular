À esquerda da Figura 1, temos colchetes envolvendo um alvo ([ngClass]) indicando que o fluxo de dados vai ser do componente para o template (data bindings).

Logo em seguida, Angular avalia a expressão getClasses() que é um método do componente retornando um string.

O método getClasses() no componente analisa se a quantidade de produtos é igual a 5

Caso true a expressão retornará "bg-success", caso false a expressão retornará bg-warning

Fluxograma do Método getClasses()

One-way data binding são recursos usados em templates HTML do Angular,
para gerar conteúdos ou modificar comportamentos através da interação com o usuário.

Possui o fluxo de dados em uma direção (Ver Figura 1):

  • Binding de Dados (Data Bindings)

    Os dados fluem do componente para o template.

  • Binding de Eventos (Event Bindings)

    Os dados fluem do template para o componente.

A estrutura básica, à esquerda da Figura 1, é dividida em 4 partes:

  • Elemento hospedeiro (Host Element)

    Elemento HTML ou tag HTML do qual vai mudar a aparência, conteúdo ou comportamento.

    Data Bindings pode ser aplicado a qualquer elemento host e um elemento host pode ter múltiplos data bindings.

  • Colchetes (Brackets)

    Colchetes (ou Parênteses) envolvem o alvo, denotando avaliar uma expressão passando o resultado para o alvo, do qual faz mudança no elemento host.

    Colchetes indicam que os dados fluem do componente para o template HTML (data bindings).

    Parênteses indicam que os dados fluem do template HTML para o componente (event bindings).

    • Obs. 1: Pode-se também usar dupla chaves visto em string interpolation bindings e uma combinação de colchetes envolvendo parênteses, chamado banana-in-box, visto em two-way bindings .

    • Obs. 2: Omitindo o colchetes ou parênteses a expressão não será avaliada, passando a ser considerada um valor literal .

  • Alvo (Target)

    É aquele que vai ser atingido pelo binding, podendo ser de 2 tipos:

    • Bindings de Propriedade (Property Bindings)

      São divididos em:

      • Binding de Propriedade Padrão (Standard Property Binding)
      • Binding de Interpolação de String (String Interpolation Binding)
      • Binding de Atributo (Attribute Binding)
      • Binding de Classe (Class Binding)
      • Binding de Estilo (Style Binding)
      • Binding de Evento (Event Binding)
    • Bindings de Diretiva (Directive Bindings)

      Podem ser fornecidos pelo Angular (built-in directives) prefixados com ng ou personalizados.

      São divididos em:

      • Diretiva NgClass
      • Diretiva NgStyle
      • Diretiva NgIf
      • Diretiva NgFor
      • Diretiva NgSwitch/NgSwitchCase/NgSwitchDefault
      • Diretiva NgTemplateOutlet
  • Expressão (Expression)

    Fragmento de código analisado pelo componente (ou escrito inline no template) através de propriedades e métodos, sendo assim o valor do alvo.

    • Obs. 1: Na expressão pode-se misturar métodos e propriedades com valor literal .

    • Obs. 2: Dependendo da complexidade da lógica não é aconselhável escrever inline no template.

    • Obs. 3: Quando se tem mais de um valor para se apresentar na expressão, deve-se separar por espaços.
