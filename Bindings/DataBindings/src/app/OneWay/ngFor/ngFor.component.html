Estrutura Básica Diretiva NgFor Binding

Usa-se quando se quer repetir a secção de conteúdos para cada objeto de um array.

Obs.: É equivalente à declaração loop foreach do JavaScript.

Obs.: Asterisco (*) indica que a diretiva é um micro-template, visto aqui .

Obs.: Usa-se geralmente para popular elementos aninhados, como as linhas de uma tabela.

Nota: Na expressão deve-se usar of em ngFor, pois existe uma diferença entre of e in da declaração for do JavaScript, visto no website .

A expressão se divide em duas partes:

  • Variável de Template (Template Variable)

    Usa-se a keyword de inicialização de variável let e o nome da variável de template (ver Figura 1).

    A variável de template representa cada objeto ou item do data source (fonte de dados) para serem usados dentro do template HTML.

    Obs.: Variáveis de template geralmente são usadas com string interpolation binding.

    Nota: Para saber mais sobre elemento variável de template consultar aqui .

    Existem outras variáveis de template (separadas por ponto-e-vígula dentro da expressão e depois do data source), são eles:

    • Index

      Indica a posição do objeto atual, sendo incrementado para cada objeto do data source.

      Obs.: O valor da variável é do tipo number.

    • Count

      Indica a quantidade de itens do data source.

      Obs.: O valor da variável é do tipo number.

    • Odd

      Indica que a posição (index) do atual objeto no data source é impar, caso retorne true.

      Obs.: É o oposto de even (visto abaixo), significando que se odd é true, even deve ser false.

      Obs.: O valor da variável é do tipo boolean.

      Obs.: A contagem da posição no data source (um array geralmente) inicia-se em zero.

    • Even

      Indica que a posição (index) do atual objeto no data source é par, caso retorne true.

      Obs.: É o oposto de odd (visto acima), significando que se even é true, odd deve ser false.

      Obs.: O valor da variável é do tipo boolean.

      Obs.: A contagem da posição no data source (um array geralmente) inicia-se em zero.

    • First

      Indica que a posição do atual objeto no data source é o primeiro, caso retorne true.

      Obs.: Quando first é true para o primeiro objeto do array, significa que todos os outros objetos vão ser false.

      Obs.: O valor da variável é do tipo boolean.

    • Last

      Indica que a posição do atual objeto no data source é o último, caso retorne true.

      Obs.: Quando last é true para o último objeto do array, significa que todos os outros objetos vão ser false.

      Obs.: O valor da variável é do tipo boolean.

  • Fonte de Dados (Data Source)

    Fornece os dados que será enumerado, sendo o array de objetos.
